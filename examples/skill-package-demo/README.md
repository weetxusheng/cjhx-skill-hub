# Skill Hub 示例技能（完整模板）

面向 **投稿人 / 内部技能维护者**：本 README 会随技能版本一并入库，并在前台技能详情中作为**主说明文档**展示。请按真实技能改写各章节。

---

## 1. 技能概述

| 项 | 说明 |
|----|------|
| **定位** | 演示「清单 + 文档 + 可选附件」的标准交付形态，非生产业务逻辑。 |
| **适用场景** | 团队首次接入 Skill Hub、需要可复制的目录与字段说明时，以此为模板裁剪。 |
| **不包含** | 本 ZIP 仅作元数据与文档示例；具体运行时指令、脚本或子目录结构由业务方自行扩展（若后续平台支持子目录解析，可再补充）。 |

---

## 2. 本 ZIP 根目录文件说明

当前约定：**根目录**必须包含下列文件（与压缩工具无关，切勿多包一层文件夹）：

| 文件 | 是否必需 | 说明 |
|------|----------|------|
| `skill.yaml` | **必需** | 机器可读清单：名称、slug、版本号、分类 slug、标签等。 |
| `README.md` | **必需** | 给人看的说明文档（即本文件）。 |
| `CHANGELOG.md` | 可选 | 版本变更记录，建议与 `skill.yaml` 的 `version` 对齐。 |

打包后解压应类似：

```text
your-skill.zip
├── skill.yaml
├── README.md
└── CHANGELOG.md   # 可选
```

---

## 3. `skill.yaml` 字段速查

| 字段 | 说明 |
|------|------|
| `name` | 展示名称，可与 slug 不同。 |
| `slug` | 全局唯一标识，建议小写与连字符；新建技能后谨慎修改。 |
| `version` | SemVer，如 `1.0.0`；同一技能下版本不可重复。 |
| `summary` | 短摘要，用于列表与卡片。 |
| `description` | 长描述，可多行；与 README 可互补。 |
| `category` | 须为后台已有**分类 slug**（非中文名）。 |
| `tags` | 字符串数组，可为空数组 `[]`。 |
| `author` | 作者或团队署名。 |

修改模板后，请同时检查 **分类 slug** 是否在环境中存在，避免上传时报「分类不存在」。

## 5. 常见问题

**Q：能否在 ZIP 里放子目录（如 `tools/`）？**  
当前上传包会整体存档；平台解析清单时以**根目录** `skill.yaml` / `README.md` 为准。子目录文件可随包保存，是否被运行环境消费以后续产品约定为准。

**Q：示例里的 slug 上传失败？**  
可能已被他人占用；请修改 `skill.yaml` 中 `slug` 与 `name` 后重新打包。

详见同目录 `CHANGELOG.md`；发版时请更新其中的日期与条目，并与 `skill.yaml` 的 `version` 一致。
