# 后台工作台说明

## 目标
后台不是“很多散乱页面”，而是一套围绕 skill 生命周期的运营工作台。

## 页面职责
### 工作台首页
- 汇总技能主档数、已发布数、待审核数、待发布数
- 提供审核中心、待发布、处理记录、技能列表等快捷入口
- 展示最近更新技能、待审核概览、待发布概览、最近处理记录
- 进入规则：具备任一工作台权限（`admin.dashboard.view` / `skill.view` / `skill.review` / `skill.publish`）即可进入首页

### 技能列表
- 查看主档、线上版本、最新版本状态、待办数量、互动统计
- 进入 skill 详情
- 上传新技能或新版本
- 必须提供常用筛选（关键字、分类、主档状态等），与 `docs/20-engineering/frontend-guide.md`「列表页」一节一致

### 审核中心
- 只处理 `submitted`
- 筛选：分类、提交人（服务端）；技能名/slug/版本关键字（本地，便于在长队列中定位）
- 重点看：提交人、分类、最近说明、审核负责人
- 动作：通过、拒绝
- 点击技能名在**弹窗**内打开版本详情（可编辑版本文案、执行通过/拒绝）；整页 `/versions/:id` 仍可从技能详情、待发布等入口使用
- 审核动作约束：
  - 通过 -> 状态转 `approved` 并进入待发布队列
  - 拒绝 -> 需填写意见并记录审计日志
  - 任一动作失败必须给出可读错误，不允许静默

### 待发布
- 只处理 `approved`
- 筛选：关键字（技能、版本、提交人等）、分类（本地，便于在长队列中定位）
- 重点看：通过时间、发布负责人、最近审核意见
- 动作：发布
- 发布动作约束：
  - 发布成功后，工作台统计和技能详情需立即反映最新线上版本
  - 发布失败要保留在待发布队列并显示原因

### 处理记录
- 查看 `approve / reject / publish / archive / rollback`
- 重点用来追溯，不作为主操作入口
- 必须可按时间倒序浏览，并可定位到具体版本详情

### Skill 详情
- 主档信息
- 版本与审核摘要
- 授权对象
- 数据统计
- 收藏/下载明细按权限展示
- 操作按钮由全局权限 + skill 授权共同决定
- 详情页模块编排：
  - 主档信息
  - 版本时间线
  - 授权对象
  - 数据统计
  - 明细记录（按权限）
- 模块间统一 16px 间距，区域块必须带稳定 `id`，便于 E2E 与排障

### 用户管理
- 查看用户角色、状态和最近登录
- 从“人”的视角查看当前用户被哪些 skill 授权
- skill 授权清单必须区分直接授权与角色继承

### 版本详情
- Manifest
- README
- 文案编辑
- 版本级动作
- 可执行动作由后端 capability 字段直接下发，不由前端自行猜测
- 状态机可执行动作示意：
  - `draft`：编辑、提审
  - `submitted`：审核通过/拒绝
  - `approved`：发布
  - `published`：归档、回滚（按权限）

## 操作与导航细节
- 左侧菜单点击必须优先尊重用户当前操作，不允许被“自动恢复上次路径”反向覆盖
- 工作区 tab 只用于提升单次会话效率，不是长期记录系统：
  - 允许会话内保留
  - 若产品要求，退出登录后必须清空
- 关闭当前 tab 后应回到最近一个有效 tab；全部关闭后回到工作台首页

## 异常与回退体验
- 接口报错时保留当前筛选与上下文，不强制整页跳转
- 权限不足时展示只读原因和下一步建议，不要整页消失
- 状态冲突（409）时提示“当前状态已变化，请刷新后重试”

## 推荐路径
- 新版本上传：技能列表 -> skill 详情 -> 上传新版本 -> 默认已进入待审核（无需单独提审）；若需改稿可在 `draft`/`rejected` 后再走「提交审核」
- 审核：审核中心 -> 弹窗内版本详情 -> 审核（或通过技能列表/版本时间线进入整页版本详情）
- 发布：待发布 -> 版本详情（可选） -> 发布
- 查询历史：处理记录 -> 版本详情
- 配权限：技能列表 -> skill 详情 -> 授权对象
