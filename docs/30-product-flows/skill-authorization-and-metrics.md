# Skill 授权与数据面

## 授权对象
- 角色授权：授予某个角色在某个 skill 上的 scope
- 指定人授权：授予某个用户在某个 skill 上的 scope

## 推荐职责
- `owner`
  - 唯一主负责人
  - 能配置授权
  - 能兜底处理审核与发布
- `maintainer`
  - 日常维护
  - 上传新版本
  - 修改草稿与被拒绝版本
- `reviewer`
  - 审核 `submitted`
- `publisher`
  - 处理 `approved`
  - 执行发布、归档、回滚
- `viewer`
  - 可查看该 skill 后台详情

## 推荐配置模板
### 小团队模式
- 1 个 owner
- 1-2 个 maintainer
- reviewer / publisher 走默认系统角色

### 业务线模式
- 每条业务线一个 owner
- 多个 maintainer
- reviewer 和 publisher 可配置为角色 + 指定人混合

## 数据面
### 汇总
- 点赞总数
- 收藏总数
- 下载总数
- 最近 7 天趋势
- 最近 30 天趋势

### 明细
- 收藏用户明细
- 下载用户/版本/IP 明细
- 未来可扩展点赞明细，但当前优先保留聚合

## 可见范围
- 管理员：
  - 可看聚合统计
  - 可看明细
- 普通运营角色：
  - 默认可看聚合
  - 明细视图按全局权限控制
  - 即使能看下载明细，也默认不看 IP 和匿名下载细节
- Portal 普通用户：
  - 只看公开聚合数据

## 口径定义
- 点赞：轻互动反馈，不代表订阅
- 收藏：进入“我的偏好”或后续查看列表
- 下载：技能包实际分发次数

## 时间窗口
- 后台默认展示最近 30 天趋势
- 需要更短窗口时，由前端做视图裁剪，不改变接口事实口径

## 刷新时机
- 点赞、收藏、下载动作成功后，前台与后台汇总值应尽快刷新
- 汇总字段与明细表不一致时，以明细表重新汇总为准

## 脱敏规则
- 非管理员默认不展示下载 IP 和匿名用户细节
- 导出时若包含用户维度信息，应遵循当前组织的数据合规要求

## 后台执行约束
- 配置 skill 授权对象时，只有 `owner` 可以新增、删除角色授权和指定人授权
- `maintainer` 可维护版本和主档，但不能修改授权对象
- Skill 详情和版本详情的按钮显隐应以“全局权限 + skill scope + 后端 capability 字段”为准
- 用户管理页应能从“人”的视角查看某用户当前命中的 skill 授权，且要区分直接授权与角色继承
