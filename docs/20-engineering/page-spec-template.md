# 页面规格模板

> **OWNER**: Skill Hub 前端负责人
>
> **Last reviewed**: 2026-03-28
>
> **Change triggers**: 新增页面 / 重构页面结构 / 新增关键抽屉与弹窗 / 交互主路径变化

## Purpose
本模板用于把页面从“概念说明”写成“可落地规格书”。任何关键页面、抽屉、工作台或高价值弹窗，都应先补或同步补一份页面规格。

## 使用规则
- 适用于：
  - 独立路由页面
  - 详情抽屉
  - 上传/审核/发布类高价值弹窗
- 不适用于：
  - 纯静态说明页
  - 无业务状态的简单空白壳页
- 命名建议：`<page-name>-spec.md`
- 页面规格应放在最接近事实来源的文档中：
  - 跨系统/产品行为：`docs/30-product-flows/`
  - 前端实现约束与模板：`docs/20-engineering/`

## 规格模板
### 1. 页面身份
- 页面名称：
- 页面类型：列表页 / 详情页 / 工作台页 / 抽屉 / 弹窗
- 目标用户：
- 路由或触发入口：

### 2. 页面目标
- 这个页面帮助用户完成什么主任务
- 这不是用来做什么的

### 3. 区块结构
按从上到下、从左到右的阅读顺序写：

| 区块 | 作用 | 必要内容 | 稳定 `id` |
|---|---|---|---|
| header | 页面摘要、标题、主操作 | 标题、状态、主按钮 | `...-header` |
| filters | 筛选区 | 筛选条件、查询、重置 | `...-filters` |
| summary | 摘要区 | 统计卡片、说明 | `...-summary` |
| list/table | 主数据区 | 列表或表格 | `...-table` / `...-list` |
| side panel | 详情或辅助信息 | 抽屉/弹窗/侧边说明 | `...-drawer` / `...-modal` |

### 4. 数据来源
| 数据 | 来源接口 | 是否首屏必需 | 刷新时机 |
|---|---|---|---|
| 页面主数据 | `GET ...` | 是 / 否 | 首屏 / 筛选变化 / 动作成功后 |
| capability | `GET ...` 或详情响应附带 | 是 / 否 | 详情刷新时 |
| 枚举/分类 | `GET ...` | 是 / 否 | 首屏 / 长缓存 |

### 5. 交互动作
| 动作 | 触发位置 | 前置条件 | 成功后 | 失败后 |
|---|---|---|---|---|
| 查询 | 筛选区 | 参数合法 | 刷新列表 | 保留筛选并显示错误 |
| 打开详情 | 行点击 / 按钮 | 目标存在且可见 | 打开抽屉/详情 | toast + 保留上下文 |
| 提交动作 | 主按钮 / 行内按钮 | capability 允许 | 刷新详情/列表/摘要 | 展示可读错误 |

### 6. 权限要求
- 页面可见需要哪些全局权限
- 页面内动作需要哪些 capability 或 skill scope
- 权限不足时显示什么，不允许页面无声消失

### 7. 状态矩阵
| 状态 | 呈现方式 | 是否保留上下文 |
|---|---|---|
| loading | 骨架屏 / Spin | 是 |
| empty | Empty + 下一步建议 | 是 |
| error | 错误提示 + 重试按钮 | 是 |
| forbidden | 只读说明 / 权限不足文案 | 是 |

### 8. 跳转与返回
- 从哪里来
- 返回后保留哪些状态：筛选、分页、滚动位置、抽屉上下文
- 自动恢复逻辑是否允许执行，执行几次

### 9. 视觉与一致性约束
- 使用哪些统一组件
- 主按钮、危险按钮、状态标签、卡片间距遵循哪份规范
- 哪些地方不允许页面自定义

### 10. 埋点 / 统计口径（如适用）
- 需要统计哪些动作
- 成功/失败是否区分
- 与后端统计口径如何对应

### 11. 测试要求
- 最低测试层级：`L1/L2/L3`
- 至少需要覆盖：
  - 首屏可打开
  - 主路径可执行
  - 权限不足态
  - 一条异常态

## 快速评审清单
- 是否明确页面目标而不是只列接口
- 是否写清区块结构与稳定 `id`
- 是否写清数据来源和刷新时机
- 是否写清 loading / empty / error / forbidden
- 是否写清权限不足时如何呈现
- 是否能指导 E2E 写出稳定断言
