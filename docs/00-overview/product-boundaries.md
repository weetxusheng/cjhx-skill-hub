# 产品边界

## 当前范围
- 企业内网使用，不做开放注册
- 人工审核 + 手工发布，不做自动机审
- 单组织、单区域部署
- 技能包以 ZIP 形式上传和下载
- 版本级 README、安装说明、Agent/Human 使用指引
- 后台支持 skill 授权、运营统计、明细追溯

## 非范围
- 多租户
- 企业 SSO
- 公网开放市场
- 在线运行执行器
- 自动恶意文件扫描
- Docker/Kubernetes 作为正式部署方式
- 用户直配全局权限

## 产品原则
- 先可运营，再做复杂自动化
- 先把状态机和权限边界做清楚，再做 UI 扩展
- 统计口径要可审计、可追溯，不能只展示冗余字段
- 工作台入口优先级高于深层详情页入口

## 明确不做的偷懒方案
- 不允许把 `approved` 直接当作线上版本
- 不允许用角色名硬编码替代权限点
- 不允许让未发布版本覆盖前台展示数据
- 不允许只做汇总、不保留统计明细来源
